                         KEY DESIGN HIGHLIGHTS
================================================================================

ARCHITECTURE:
  ‚Ä¢ Master Nodes (Go):
    - Raft-based cluster state management
    - Shard allocation service
    - Index metadata management
    
  ‚Ä¢ Coordination Nodes (Go + Python):
    - DSL/PPL query parsing
    - Apache Calcite optimization
    - Python pipeline execution
    - Result aggregation
    
  ‚Ä¢ Data Nodes (C++ Diagon):
    - Inverted Index: Text search with SIMD BM25 (4-8√ó faster)
    - Forward Index: ClickHouse columnar storage with skip indexes
    - Computation: Joins, ML inference, Python UDFs

TECHNOLOGY STACK:
  ‚Ä¢ Languages: Go (orchestration), C++ (data nodes), Python (pipelines), 
               Java (Calcite)
  ‚Ä¢ Orchestration: Kubernetes with custom operator
  ‚Ä¢ Storage: S3/MinIO/Ceph for cold tier, NVMe/SSD for hot/warm
  ‚Ä¢ Monitoring: Prometheus + Grafana + OpenTelemetry
  ‚Ä¢ Consensus: Raft (via etcd/raft library)

PERFORMANCE TARGETS:
  ‚Ä¢ Indexing: 100k docs/sec/node
  ‚Ä¢ Query Latency (p99): <10ms (term), <50ms (boolean), <100ms (aggregation)
  ‚Ä¢ SIMD Speedup: 4-8√ó for BM25 scoring
  ‚Ä¢ Storage: 40-70% smaller (3-5√ó compression ratio)
  ‚Ä¢ Scale: 1000+ nodes, 10+ TB per index

COST SAVINGS:
  ‚Ä¢ Infrastructure: 25-42% reduction vs OpenSearch
  ‚Ä¢ 100M docs: $49k/year (Quidditch) vs $71k/year (OpenSearch) = $22k savings
  ‚Ä¢ 1B docs: $383k/year (Quidditch) vs $513k/year (OpenSearch) = $130k savings
  ‚Ä¢ Payback Period: 8-12 months
